## 3. Core workflow rules

### 3.1 вЂ” Backup, commit, structure-update
1. Backup before editing `ECT_preprint.tex`/`ECT_companion.tex` в†’ `backup/ECT_preprint_BACKUP_vN_<desc>.tex`
2. New references в†’ `references.bib` immediately
3. After confirmed edit в†’ git commit + push
4. Structure changes в†’ update THIS FILE immediately
5. Major restructuring в†’ draft in `proposals/` first, get approval, then implement
6. Page count rule вЂ” after every compilation, report "X pages (was Y)"
7. Edit permission вЂ” NEVER edit preprint without explicit "integrate"/"insert"/"РІРЅРµРґСЂСЏР№" approval
8. PHENOM-RULE вЂ” unsanctioned phenomenological numbers: flag, don't propagate

### 3.2 вЂ” Figure rules
- рџЋЁ **All figures/plots MUST be grayscale**. Use linestyles (solid/dashed/dotted/dash-dot) and markers (circle/square/triangle/diamond).
- Filenames: `fig_<topic>.{pdf,png}`. Scripts: `scripts/fig_<topic>.py`.

### 3.3 вЂ” Appendix insertion order rule
- New appendices inserted in **order of first reference**. Rename existing letters if needed. Apply independently to companion.

### 3.4 вЂ” Language discipline
- Never "derived" unless rigorously proven in current paper
- Never "ECT predicts X" when only "ECT admits a route toward X" is justified
- "Effective" framing preferred for data-inferred quantities
- "Level A / B / Open" вЂ” strict usage
- **Cosmological Оµ** вЂ” see В§6.3 language rules v5

### 3.5 вЂ” Principle of no information loss (3 axes)

**Axis A вЂ” migration disposition:** verbatim / reframe / demote-to-appendix / retire
**Axis B вЂ” info-loss level:** content / structural (labels, refs, fig/tab/app) / argument-flow
**Axis C вЂ” Can this statement survive unchanged?** yes / rephrase / relocate / remove

No silent drops. Every existing item gets A Г— B Г— C assignment in Step 2.0 inventory.

### 3.6 вЂ” Parallel creation strategy for major restructurings

- **Phase 1 вЂ” create in parallel.** New subsections (В§15.5'), new appendices alongside existing. Old content NOT touched.
- **Phase 2 вЂ” validate.** Compile, cross-check new vs old, GPT review on both.
- **Phase 3 вЂ” switch.** Single session: remove old, update all cross-refs in one pass.
- **Phase 4 вЂ” cleanup.** Reports, audits, companion mirror.
